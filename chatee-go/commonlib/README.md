# CommonLib 模块文档规范

## 文档约定

### 模块文档要求

**每个功能模块必须在其目录下包含 `README.md` 文件**，用于说明：

1. **技术细节**：实现原理、架构设计、技术选型等
2. **业务细节**：业务逻辑、使用场景、业务流程等
3. **使用示例**：代码示例、最佳实践等
4. **注意事项**：常见问题、限制、注意事项等

### 文档维护规则

**每次修改业务模块后，必须同步更新对应的 `README.md`**，确保文档与代码保持一致。

### 模块划分原则

- **按功能模块划分**：每个独立的功能模块应该有独立的目录
- **模块职责清晰**：每个模块应该有明确的职责边界
- **文档与代码同步**：代码变更时，文档必须同步更新

## README.md 模板

每个模块的 README.md 应该包含以下部分：

```markdown
# 模块名称

## 概述
简要说明模块的用途和功能

## 目录结构
列出模块的文件结构

## 核心概念
说明模块的核心概念和设计思路

## 使用方式
提供使用示例和代码示例

## 技术细节
详细说明实现原理和技术选型

## 业务细节
说明业务逻辑和使用场景（如适用）

## 注意事项
列出常见问题、限制和注意事项

## 参考
相关文档和资源链接
```

## 文档更新检查清单

修改模块代码时，检查以下内容是否需要更新：

- [ ] 新增功能是否在 README 中说明
- [ ] API 变更是否在 README 中更新
- [ ] 使用示例是否仍然正确
- [ ] 技术细节是否反映最新实现
- [ ] 注意事项是否需要更新
